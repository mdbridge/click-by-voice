# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working
with code in this repository.


## Architecture

This is a Chrome Manifest V3 (MV3) extension.  Source lives entirely in `src/`.

### Module system

- **Content scripts** (`src/content_scripts/`): Hand-rolled modules
  (IIFE pattern).  MV3 does not support ES6 modules in content
  scripts.  See `utilities.js` for the conventions.

- **Background, popup, options**: ES6 modules.  Mark exports with the
  `export` keyword directly on the function/variable declaration.

- Main entry files (`background.js`, `content_script.js`) call other
  files; other files should not call the main files.

### Content script -> Background communication

All content script requests go through `Util.act` (fire-and-forget) and
`Util.request` (returns a promise) defined in `utilities.js`.  Both
automatically attach `args.epoch`.

### Epoch system

Epochs are generated by the background service worker
(`background_persistence.js`) per tab session.  Content scripts learn
their epoch via `CBV_NEW_EPOCH` messages, stored in `current_epoch`
inside the `Util` IIFE.  Epoch checking happens in the **service
worker** (not content scripts) because another frame may have advanced
the epoch.  Background checks epoch via
`background_utilities.epoch_is_current(tab_id, epoch)` at the top of
`handle_content_script_message`.  `CBV_HELLO` is exempt from epoch
checking.

### Error response convention

- `{ rejected: true }` -- stale epoch rejection
- `{ error: "..." }`   -- unexpected error

### Code conventions

- `// <<<>>>` comments mark items needing review/consideration -- do not remove them
- Inside content scripts, use `Util.vlog(level)(message)` for verbose logging
- Git commands: use `git -C $HOME/infra/browser_extensions/click_by_voice`
  prefix if needed
- each major component is in a separate subdirectory, with a README.txt
  file that you may find helpful.


## Building

```bash
make              # packages extension into click_by_voice.zip
make clean        # removes temporary files
```

The build strips comments from `manifest.json` and zips the `src/`
directory.  There is no npm, webpack, or transpilation step -- this is
plain JavaScript loaded directly by Chrome.


## Testing

Testing is manual via the HTML files in `test_pages/`.  Load the
unpacked extension in Chrome (`chrome://extensions/` -> Load unpacked ->
select `src/`), then open test pages to verify behavior.  There is no
automated test runner.

### Syntax checking

After editing JavaScript files, syntax-check them with node's
parse-only mode.  On Windows, node is not on the Bash tool's PATH, so
invoke it by full path:

  '/c/Program Files/nodejs/node.exe' --check <file.js>

On Linux, plain `node --check <file.js>` works.


## Running shell commands

On **Linux**, run shell commands normally.

On **Windows**, `make` and similar Unix tools require Cygwin bash
(exception: git -- see the Git section below).  Always invoke it like
this using the PowerShell tool:

  & "C:/cygwin64/bin/bash.exe" -lc '<command>'

**Avoid double quotes inside the `-lc` string.**  PowerShell 5.1 mangles
double quotes when passing arguments to native executables, so any
double quote inside the `-lc` string gets eaten before bash sees it --
the message gets truncated or lost entirely.

If any argument requires quoting (e.g., a grep pattern with spaces),
write the value to a temporary file then reference that file in the
command instead.

Before using Cygwin bash, you MUST determine what directory `~` refers
by running `echo ~` in either bash.  Never simply assume that `~` refers
to the Windows user profile directory.

**`/cygdrive/` Cygwin paths are intermediate only.  Never use them as a
final path or in any command -- always convert to `~/...` form before
use.**

### Temporary files

If you need a temporary file, put it in the top directory of the
repository ending in a `~`; e.g., `command_output~`.  Do not put
temporary files in `/tmp` .

### Git

On Windows, always run git commands using the Bash tool (which is Git
Bash, i.e., Git for Windows).  NEVER run git through Cygwin bash or the
PowerShell tool.
